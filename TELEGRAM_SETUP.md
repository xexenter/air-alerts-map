# Telegram-бот повітряних тривог — налаштування

Все живе в тому самому Vercel-проєкті, що й карта. Окремий сервер не
потрібен: Telegram Bot API — це звичайні HTTP-запити, тому serverless
підходить ідеально.

Що з'являється:
- **Сповіщення в чат/канал** при появі чи зникненні тривоги в області.
- **Команда `/status`** — миттєвий поточний список тривог прямо в Telegram.

---

## Крок 1. Створити бота

1. У Telegram знайти **@BotFather** → написати `/newbot`.
2. Дати ім'я і username (username має закінчуватись на `bot`).
3. BotFather дасть токен виду `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxx`
   — це `TELEGRAM_BOT_TOKEN`.

## Крок 2. Отримати chat_id, куди слати сповіщення

Варіант A — **особистий чат із собою/групою**:
1. Написати новоствореному боту що-небудь (напр. `/start`) в особисті.
2. Відкрити в браузері (підставивши свій токен):
   `https://api.telegram.org/bot<ТВІЙ_ТОКЕН>/getUpdates`
3. У відповіді знайти `"chat":{"id":123456789,...}` — це і є `TELEGRAM_CHAT_ID`.

Варіант B — **канал** (зручніше, якщо сповіщення мають бачити інші):
1. Створити Telegram-канал (публічний або приватний).
2. Додати бота туди адміністратором (потрібні права хоча б на публікацію повідомлень).
3. Якщо канал публічний — `TELEGRAM_CHAT_ID` це просто `@назва_каналу`.
   Якщо приватний — треба переслати будь-яке повідомлення з каналу боту
   @userinfobot, він покаже numeric ID каналу (виду `-1001234567890`).

## Крок 3. Підключити Upstash Redis (для зберігання стану між перевірками)

Serverless-функція не пам'ятає нічого між викликами, тому попередній стан
тривог (щоб розуміти, що саме ЗМІНИЛОСЬ) зберігаємо в Redis.

1. У дашборді проєкту на Vercel: **Storage → Create Database → Redis (Upstash)**.
   (Якщо побачиш просто "Marketplace Database Integrations" — обери там Upstash.)
2. Обери безкоштовний план, регіон — будь-який ближче до тебе.
3. Підключи цю базу до свого проєкту (`Connect Project`) — Vercel сам додасть
   потрібні змінні оточення (`KV_REST_API_URL`, `KV_REST_API_TOKEN` або
   `UPSTASH_REDIS_REST_URL`/`TOKEN` — бібліотека в коді підхоплює будь-який
   з цих варіантів автоматично).

## Крок 4. Додати змінні оточення в Vercel

Settings → Environment Variables, додати (окрім тих, що вже додав Upstash):

| Name | Value |
|---|---|
| `TELEGRAM_BOT_TOKEN` | токен з кроку 1 |
| `TELEGRAM_CHAT_ID` | chat_id/`@channel` з кроку 2 |
| `CRON_SECRET` | будь-який довгий випадковий рядок (сам придумай) — захищає `/api/telegram-check` від чужих викликів |

Після додавання — Redeploy (Deployments → "..." → Redeploy), щоб змінні підхопились.

## Крок 5. Зареєструвати webhook (щоб працювала команда /status)

Один раз виконати (підставивши свої значення) — просто відкрити цей URL
у браузері:

```
https://api.telegram.org/bot<ТВІЙ_ТОКЕН>/setWebhook?url=https://<твій-сайт>.vercel.app/api/telegram-webhook
```

У відповіді має бути `"ok":true`. Тепер напиши боту `/status` в Telegram —
має одразу відповісти поточним списком тривог.

## Крок 6. Налаштувати регулярну перевірку (для сповіщень про ЗМІНИ)

Vercel Hobby дозволяє власний Cron не частіше разу на добу — для сповіщень
про тривогу цього замало. Рішення — безкоштовний зовнішній планувальник,
який просто стукає на твій URL кожну хвилину-дві:

1. Зареєструватись на https://cron-job.org (безкоштовно, без картки).
2. Create cronjob:
   - URL: `https://<твій-сайт>.vercel.app/api/telegram-check`
   - Schedule: кожну 1 хвилину (або 2, якщо хочеш рідше)
   - **Additional headers**: додати заголовок
     `X-Cron-Secret: <той самий CRON_SECRET, що в Vercel>`
3. Зберегти. Перший виклик просто запам'ятає поточний стан (не спамитиме
   вже активними тривогами), а всі наступні — сповіщатимуть тільки про
   ЗМІНИ.

Перевірити вручну, що ендпоінт взагалі живий:
```bash
curl -H "X-Cron-Secret: твій_секрет" https://<твій-сайт>.vercel.app/api/telegram-check
```
Має повернути JSON з `"ok":true`.

---

## Як це узгоджено з картою

`/api/alerts` (карта) і `/api/telegram-check` (бот) використовують одну й
ту саму функцію `getActiveOblasts()` з `api/lib/getActiveOblasts.js` —
тобто дані завжди ідентичні, розсинхрону бути не може.

## Що можна додати далі

- **Кілька підписників на різні області** — зберігати в тому ж Redis
  мапу `chat_id → список областей`, і в `/api/telegram-check` слати
  персоналізовані повідомлення тільки тим, хто підписаний на конкретну
  область (команда `/subscribe Полтавська` в `telegram-webhook.js`).
- **Частіші перевірки** — cron-job.org безкоштовно тягне і раз на хвилину;
  якщо треба ще частіше — платні тарифи або власний невеликий сервер.
