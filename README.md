# Карта повітряних тривог — деплой

Готовий проєкт: статична карта (`public/index.html`) + серверна функція-проксі
(`api/alerts.js`), яка ховає токен alerts.in.ua від браузера.

Нижче — покроковий шлях **без свого сервера**, через Vercel (безкоштовно для
такого проєкту) і власний домен.

---

## Крок 1. Отримати токен alerts.in.ua

1. Відкрити https://alerts.in.ua/api-request і заповнити форму (коротко
   опишіть проєкт: "карта тривог для людей, некомерційний").
2. Токен приходить на пошту, зазвичай не миттєво — можна робити наступні
   кроки паралельно, сайт і без токена відкриється (просто покаже банер
   "токен не налаштовано" замість реальних тривог).

## Крок 2. Викласти код на GitHub

1. Зареєструватися на https://github.com, якщо ще немає акаунту.
2. Створити новий репозиторій (кнопка "New repository"), наприклад
   `air-alerts-map`.
3. Залити туди всі файли з цієї папки (`public/`, `api/`, `package.json`,
   `vercel.json`) — простіше за все перетягнути їх у вікно
   "uploading an existing file" на сторінці репозиторію на GitHub.

## Крок 3. Задеплоїти на Vercel

1. Зайти на https://vercel.com і зареєструватися **через GitHub**
   (це одразу пов'яже акаунти).
2. Натиснути "Add New" → "Project", обрати щойно створений репозиторій
   `air-alerts-map`.
3. Нічого не змінюючи в налаштуваннях білду, натиснути "Deploy".
4. Через хвилину Vercel видасть посилання типу
   `https://air-alerts-map-xxxx.vercel.app` — це вже робочий сайт.

## Крок 4. Додати токен у налаштування Vercel

1. У проєкті на Vercel: Settings → Environment Variables.
2. Додати змінну:
   - Name: `ALERTS_TOKEN`
   - Value: токен, який прийшов з alerts.in.ua
3. Натиснути Save, потім зайти в Deployments → на останньому деплої
   відкрити меню "..." → Redeploy (щоб змінна підхопилась).

Після цього на сайті замість банера "токен не налаштовано" з'явиться
"Реальні дані · оновлено HH:MM:SS", і області почнуть фарбуватись за
справжніми тривогами.

## Крок 5. Підключити свій домен

1. Купити домен. Найпростіше — https://www.namecheap.com або
   https://domains.cloudflare.com (там же можна й зареєструвати,
   і одразу керувати DNS). Українська зона `.ua` теж підходить, якщо
   хочеться саме її — реєструється через акредитованих реєстраторів
   (список на https://hostmaster.ua).
2. У Vercel: Settings → Domains → вписати куплений домен → Add.
3. Vercel покаже, які DNS-записи додати (зазвичай один `A`-запис на
   IP Vercel і один `CNAME` для `www`). Ці записи додаються в панелі
   керування DNS у того, де куплений домен (Namecheap / Cloudflare тощо).
4. DNS оновлюється від кількох хвилин до кількох годин. Коли готово,
   Vercel сам видасть безкоштовний HTTPS-сертифікат — нічого додатково
   робити не треба.

---

## Що можна покращити далі

- **Кешування на сервері довше під час затишшя** — зараз кеш 15 секунд
  завжди; можна зробити розумніше (рідше опитувати вночі).
- **Push-сповіщення** в браузері, коли з'являється тривога в обраній
  користувачем області (Web Push API) — наступний логічний крок для
  "живого" застосунку.
- **Реальні позиції цілей (шахедів/ракет)** — на відміну від тривог по
  областях, публічного API з координатами цілей у реальному часі не
  існує. Це поточна секція карти лишається демонстраційною симуляцією;
  щоб зробити її реальною, потрібно парсити повідомлення моніторингових
  Telegram-каналів і геокодувати згадані населені пункти — це вже окремий,
  набагато складніший проєкт.
